Smart Home

Đồ án tốt nghiệp © 2025

Tổng quan

Smart Home là hệ thống điều khiển nhà thông minh hoàn chỉnh, kết hợp ESP32 và Django để tạo giải pháp IoT thực tế:

  • Điều khiển thiết bị từ xa qua web
  • Giám sát cảm biến realtime
  • Nhận diện khuôn mặt mở cửa
  • Cảnh báo khẩn cấp gas/cháy
  • Tự động bật đèn khi có người
  • Giao thức MQTT chuẩn IoT

Cơ sở dữ liệu (Database Schema)

Hệ thống sử dụng SQLite/PostgreSQL với 7 bảng chính:

1. Profile (Hồ sơ người dùng)

Mở rộng User mặc định của Django, lưu thông tin cá nhân và API Key

Trường Kiểu dữ liệu Mô tả
idInteger (PK)Khóa chính tự tăng
userForeignKey → UserLiên kết 1-1 với User Django
avatarImageFieldẢnh đại diện
birthdayDateFieldNgày sinh
genderSmallIntegerGiới tính (1=Nam, 2=Nữ)
phoneCharField(32)Số điện thoại
addressCharField(255)Địa chỉ
api_keyCharField(64)API Key để kết nối ESP32 qua MQTT
created_atDateTimeNgày tạo
updated_atDateTimeNgày cập nhật
2. Device (Thiết bị)

Lưu thông tin thiết bị smart home: Relay (bật/tắt) và Sensor (cảm biến)

Trường Kiểu dữ liệu Mô tả
idInteger (PK)Khóa chính
userForeignKey → UserChủ sở hữu thiết bị
device_codeCharField(10)Mã thiết bị: v1, v2, v3... (Unique per user)
nameCharField(100)Tên thiết bị (VD: Đèn phòng khách)
device_typeCharField(20)relay (bật/tắt) | sensor (cảm biến)
valueCharField(50)Relay: 0/1 | Sensor: giá trị đo
unitCharField(20)Đơn vị: °C, %, ppm...
is_onlineBooleanTrạng thái online/offline
enable_loggingBooleanBật ghi log để vẽ đồ thị
created_atDateTimeNgày tạo
updated_atDateTimeNgày cập nhật
3. SensorLog (Lịch sử cảm biến)

Lưu trữ giá trị cảm biến theo thời gian để vẽ biểu đồ (mỗi 5 phút)

Trường Kiểu dữ liệu Mô tả
idInteger (PK)Khóa chính
deviceForeignKey → DeviceThiết bị cảm biến
valueFloatGiá trị đo được (làm tròn 1 chữ số thập phân)
timestampDateTimeThời điểm ghi nhận
4. FaceUser (Người dùng Face ID)

Người đăng ký nhận diện khuôn mặt để mở cửa

Trường Kiểu dữ liệu Mô tả
idInteger (PK)Khóa chính
user_codeCharField(50)Mã người dùng (Unique)
nameCharField(100)Họ tên
ownerForeignKey → UserChủ nhà (quản lý)
is_activeBooleanĐang hoạt động
created_atDateTimeNgày tạo
5. FaceImage (Ảnh khuôn mặt)

Lưu đường dẫn ảnh khuôn mặt của mỗi người dùng

Trường Kiểu dữ liệu Mô tả
idInteger (PK)Khóa chính
face_userForeignKey → FaceUserNgười dùng Face ID
image_pathCharField(500)Đường dẫn file ảnh
created_atDateTimeNgày tạo
6. AttendanceLog (Lịch sử nhận diện)

Ghi lại mỗi lần nhận diện thành công (mở cửa)

Trường Kiểu dữ liệu Mô tả
idInteger (PK)Khóa chính
face_userForeignKey → FaceUserNgười được nhận diện
event_typeCharField(20)checkin | checkout | check
confidenceFloatĐộ tin cậy (0.0 - 1.0)
timestampDateTimeThời gian nhận diện
7. FaceRecognitionStatus (Trạng thái gần nhất)

Lưu trạng thái nhận diện gần nhất của mỗi người (1-1 với FaceUser)

Trường Kiểu dữ liệu Mô tả
idInteger (PK)Khóa chính
face_userOneToOne → FaceUserLiên kết 1-1
last_eventCharField(20)Sự kiện cuối cùng
last_timeDateTimeThời gian cuối cùng
Quan hệ giữa các bảng:
  • User ↔ Profile (1:1) - Mỗi user có 1 profile
  • User → Device (1:N) - User có nhiều thiết bị
  • Device → SensorLog (1:N) - Sensor có nhiều log
  • User → FaceUser (1:N) - Chủ nhà quản lý nhiều người Face ID
  • FaceUser → FaceImage (1:N) - Mỗi người có nhiều ảnh
  • FaceUser → AttendanceLog (1:N) - Mỗi người có nhiều lịch sử
  • FaceUser ↔ FaceRecognitionStatus (1:1) - Trạng thái gần nhất

MQTT Topics

# ESP32 gửi dữ liệu lên Server (receiver)
apikey/{API_KEY}/receiver/v1    → Trạng thái đèn: {"status": "on/off"}
apikey/{API_KEY}/receiver/v2    → Trạng thái cửa: {"status": "on/off"}
apikey/{API_KEY}/receiver/v3    → Trạng thái quạt: {"status": "on/off"}
apikey/{API_KEY}/receiver/v4    → Lửa: {"value": 0/1}
apikey/{API_KEY}/receiver/v5    → Gas: {"value": 0-4095}
apikey/{API_KEY}/receiver/v6    → Độ ẩm: {"value": 0-100}
apikey/{API_KEY}/receiver/v7    → Nhiệt độ: {"value": -40~85}

# Server gửi lệnh xuống ESP32 (control)
apikey/{API_KEY}/control/v1     → {"value": "on/off"}
apikey/{API_KEY}/control/v2     → {"value": "on/off/door_temp"}  ← door_temp = mở 5s
apikey/{API_KEY}/control/v3     → {"value": "on/off"}

Tính năng đặc biệt

Emergency Mode
Khi Gas ≥ 300 hoặc phát hiện lửa → BẬT TẤT CẢ relay (đèn báo, quạt thông gió, mở cửa thoát hiểm)
Face → Door
Nhận diện thành công → MQTT door_temp → Cửa mở 5 giây rồi tự đóng. Interval: 15 giây chống spam.
PIR Auto Light
Cảm biến PIR phát hiện chuyển động → Bật đèn → Sau 30 giây không có người → Tắt đèn.
Sync on Connect
ESP32 kết nối/reconnect MQTT → Tự động gửi trạng thái tất cả relay → Dashboard luôn đồng bộ.

Hướng dẫn sử dụng

  1. Đăng ký / Đăng nhập - Tạo tài khoản mới hoặc đăng nhập
  2. Lấy API Key - Vào "Hồ sơ & API Key" để copy API Key
  3. Cấu hình ESP32 - Nạp code Arduino với API Key của bạn
  4. Thêm thiết bị - Dashboard → "Thêm thiết bị" với Device Code v1-v7
  5. Điều khiển - Bật/tắt thiết bị, xem biểu đồ cảm biến realtime
  6. Face ID (tùy chọn) - Đăng ký khuôn mặt để mở cửa tự động
