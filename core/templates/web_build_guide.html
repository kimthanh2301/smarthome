Thiết kế Database (Models)

Database là gì? Tại sao cần?

Database (cơ sở dữ liệu) giống như một cuốn sổ ghi chép của hệ thống:
  • 📝 Ghi nhớ ai đăng nhập (User)
  • 💡 Ghi nhớ có những thiết bị nào, đang bật hay tắt (Device)
  • 🌡️ Ghi nhớ lịch sử nhiệt độ theo thời gian (SensorLog)
  • 👤 Ghi nhớ khuôn mặt ai được mở cửa (FaceUser)

Nếu không có database, mỗi lần tắt máy tính → mất hết dữ liệu!
2.1. Cách tư duy: Từ yêu cầu → Bảng dữ liệu

Hãy nghĩ đơn giản: Mỗi "thứ" cần lưu trữ = 1 bảng (table)

Yêu cầu hệ thống
  1. Người dùng đăng nhập
  2. Quản lý thiết bị (đèn, quạt...)
  3. Xem lịch sử cảm biến
  4. Nhận diện khuôn mặt mở cửa
Bảng cần tạo
  1. User + Profile
  2. Device
  3. SensorLog
  4. FaceUser + FaceImage

2.3. Mối quan hệ giữa các bảng
Quy tắc đơn giản:
  • 1 — 1: Một User có đúng một Profile
  • 1 — n: Một User có nhiều Device, nhưng mỗi Device chỉ thuộc 1 User
📁 User (Tài khoản đăng nhập) │ ├── 1:1 ─── Profile (Thông tin + API Key) │ ├── 1:n ─── Device (Danh sách thiết bị) │ └── 1:n ─── SensorLog (Lịch sử) │ └── 1:n ─── FaceUser (Người được mở cửa) ├── 1:n ─── FaceImage (Ảnh khuôn mặt) └── 1:n ─── AttendanceLog (Lịch sử mở cửa)
2.4. Viết code Model trong Django

Trong Django, mỗi bảng = 1 class Python. Django tự động tạo bảng SQL từ class:

# File: devices/models.py from django.db import models from django.contrib.auth.models import User class Device(models.Model): # Liên kết đến User (1 User có nhiều Device) user = models.ForeignKey( User, on_delete=models.CASCADE, # Xóa User → xóa luôn Device related_name='devices' # user.devices.all() để lấy danh sách ) # Các trường dữ liệu device_code = models.CharField(max_length=10) # "v1", "v2"... name = models.CharField(max_length=100) # "Đèn phòng khách" device_type = models.CharField(max_length=20) # "relay" hoặc "sensor" value = models.CharField(default='0') # "0"=tắt, "1"=bật # Ràng buộc: mỗi user không được trùng device_code class Meta: unique_together = ['user', 'device_code']
2.5. Chạy lệnh tạo Database

Sau khi viết xong các file models.py, chạy các lệnh sau để Django tạo database:

Bước 1: Tạo file migration (bản thiết kế)
python manage.py makemigrations

Lệnh này đọc tất cả models.py và tạo file "migration" mô tả cần tạo bảng gì.

Bước 2: Áp dụng vào database (tạo bảng thật)
python manage.py migrate

Lệnh này đọc file migration và thực sự tạo các bảng trong database (file db.sqlite3).

Bước 3: Tạo tài khoản admin (lần đầu)
python manage.py createsuperuser

Tạo tài khoản admin để đăng nhập vào /admin/ quản lý dữ liệu.

Lưu ý quan trọng:
  • Mỗi lần sửa models.py (thêm/xóa/sửa trường) → phải chạy lại makemigrations + migrate
  • File db.sqlite3 chứa toàn bộ dữ liệu. Xóa file này = mất hết!
  • Khi deploy lên server, thường dùng PostgreSQL thay vì SQLite
Kết luận Phần 2:
  • ✅ Hiểu database là "cuốn sổ ghi nhớ" của hệ thống
  • ✅ Biết cách tư duy: yêu cầu → bảng → cột
  • ✅ Hiểu mối quan hệ 1-1, 1-n giữa các bảng
  • ✅ Biết viết Django Model và chạy lệnh tạo database

Xây dựng Backend (Views, URLs)

Backend là gì?

Backend giống như bộ não của hệ thống:
  • 🧠 Nhận yêu cầu từ trình duyệt (User click nút)
  • 🧠 Xử lý logic (đọc/ghi database, gửi lệnh MQTT)
  • 🧠 Trả về kết quả (HTML, JSON, redirect...)

Không có backend → Web chỉ là trang tĩnh, không tương tác được!
3.1. Hiểu về URL và View
URL là gì?

URL = địa chỉ trang web

  • /dashboard/ → Trang dashboard
  • /devices/ → Danh sách thiết bị
  • /api/toggle/v1/ → Bật/tắt thiết bị v1
View là gì?

View = hàm Python xử lý URL đó

  • Nhận request từ user
  • Xử lý logic nghiệp vụ
  • Trả về response (HTML/JSON)

3.2. Xác định các chức năng cần làm

Từ yêu cầu hệ thống, liệt kê các hành động user có thể làm:

Chức năng URL View cần tạo Trả về
Xem dashboard /smart-home/ smart_home_page() HTML trang dashboard
Xem danh sách thiết bị /devices/ device_list() HTML danh sách
Thêm thiết bị mới /devices/add/ device_add() Form + redirect
Bật/tắt thiết bị /api/device/toggle/ api_toggle_device() JSON {status: ok}

Quy tắc đặt tên:
  • URL bắt đầu /api/ → trả về JSON (cho JavaScript gọi)
  • URL thường → trả về HTML (trang web)
  • Tên view nên mô tả rõ chức năng: device_list, device_add, device_delete
3.3. Viết code URLs (urls.py)

File urls.py là "bản đồ" kết nối URL với View:

# File: devices/urls.py from django.urls import path from . import views urlpatterns = [ # Trang HTML path('devices/', views.device_list, name='device_list'), path('devices/add/', views.device_add, name='device_add'), path('devices/<int:pk>/edit/', views.device_edit, name='device_edit'), path('devices/<int:pk>/delete/', views.device_delete, name='device_delete'), # API trả về JSON path('api/device/toggle/', views.api_toggle_device, name='api_toggle'), path('api/sensor/<int:device_id>/data/', views.api_sensor_data, name='api_sensor'), ]
Giải thích cú pháp:
  • <int:pk> = tham số động, VD: /devices/5/edit/ → pk=5
  • name='device_list' = đặt tên để dùng trong template: {% templatetag openblock %} url 'device_list' {% templatetag closeblock %}
3.4. Viết code Views (views.py)

Mỗi View là 1 hàm nhận request và trả về response:

View trả về HTML (trang web)
# File: devices/views.py from django.shortcuts import render, redirect, get_object_or_404 from django.contrib.auth.decorators import login_required from .models import Device @login_required # Bắt buộc đăng nhập mới vào được def device_list(request): """Hiển thị danh sách thiết bị của user đang đăng nhập""" # Bước 1: Lấy dữ liệu từ database devices = Device.objects.filter(user=request.user) # Bước 2: Truyền dữ liệu vào template HTML context = { 'devices': devices, 'total': devices.count(), } # Bước 3: Render và trả về HTML return render(request, 'devices/device_list.html', context)
View xử lý Form (thêm/sửa)
@login_required def device_add(request): """Thêm thiết bị mới""" if request.method == 'POST': # User đã submit form → lấy dữ liệu và lưu name = request.POST.get('name') device_code = request.POST.get('device_code') device_type = request.POST.get('device_type') # Tạo mới trong database Device.objects.create( user=request.user, name=name, device_code=device_code, device_type=device_type ) # Redirect về trang danh sách return redirect('device_list') # GET request → hiển thị form trống return render(request, 'devices/device_add.html')
View API trả về JSON
from django.http import JsonResponse from django.views.decorators.http import require_POST @login_required @require_POST # Chỉ chấp nhận POST request def api_toggle_device(request): """API bật/tắt thiết bị - gọi từ JavaScript""" # Bước 1: Lấy dữ liệu từ request device_code = request.POST.get('device_code') action = request.POST.get('action') # 'on' hoặc 'off' # Bước 2: Tìm thiết bị trong database try: device = Device.objects.get( user=request.user, device_code=device_code ) except Device.DoesNotExist: return JsonResponse({'error': 'Device not found'}, status=404) # Bước 3: Cập nhật giá trị device.value = '1' if action == 'on' else '0' device.save() # Bước 4: Gửi lệnh qua MQTT đến ESP32 send_mqtt_command(request.user, device_code, action) # Bước 5: Trả về JSON return JsonResponse({ 'status': 'ok', 'device': device_code, 'value': device.value })

Giao tiếp MQTT với ESP32

MQTT là gì? Tại sao không dùng HTTP?

MQTT (Message Queuing Telemetry Transport) là giao thức nhắn tin nhẹ, thiết kế cho IoT:
  • 📦 Nhẹ: Header chỉ 2 bytes (HTTP: hàng trăm bytes)
  • ⚡ Realtime: ESP32 nhận lệnh ngay lập tức (HTTP phải poll liên tục)
  • 🔌 Tiết kiệm pin: Giữ kết nối mở, không cần handshake mỗi lần
  • 📡 Pub/Sub: Nhiều thiết bị nhận cùng 1 message
4.1. Mô hình Publish/Subscribe

MQTT hoạt động theo mô hình Pub/Sub (Xuất bản/Đăng ký):

MQTT Broker
Trung gian

4.2. Thiết kế Topic

Topic giống như "địa chỉ" của message. Cần thiết kế rõ ràng:

# Cấu trúc Topic của hệ thống # Topic điều khiển thiết bị (Django → ESP32) {api_key}/{device_code} VD: abc123xyz/v1 # Điều khiển relay v1 của user có api_key "abc123xyz" VD: abc123xyz/v2 # Điều khiển relay v2 # Topic cảm biến (ESP32 → Django) {api_key}/{device_code}/sensor VD: abc123xyz/v5/sensor # Gửi dữ liệu nhiệt độ từ cảm biến v5 # Topic trạng thái (ESP32 → Django) {api_key}/status VD: abc123xyz/status # ESP32 báo online/offline
Tại sao dùng API Key trong topic?
  • 🔐 Mỗi user có topic riêng, không nhầm lẫn
  • 🏠 User A bật đèn không ảnh hưởng User B
  • 🔑 API Key được tạo random, khó đoán
4.3. Code MQTT Client (Django)

File mqtt/mqtt_client.py - Kết nối và gửi message:

# File: mqtt/mqtt_client.py import paho.mqtt.client as mqtt import json import threading # Cấu hình MQTT Broker MQTT_BROKER = "103.252.136.205" MQTT_PORT = 1883 MQTT_USER = "smarthome" MQTT_PASS = "password123" # Biến global lưu client mqtt_client = None def on_connect(client, userdata, flags, rc): """Callback khi kết nối thành công""" if rc == 0: print("✅ Connected to MQTT Broker") # Subscribe các topic cần lắng nghe client.subscribe("#") # Nhận tất cả message else: print(f"❌ Connection failed: {rc}") def on_message(client, userdata, msg): """Callback khi nhận message từ ESP32""" topic = msg.topic payload = msg.payload.decode() print(f"📩 Received: {topic} → {payload}") # Xử lý message (VD: lưu sensor data vào DB) if "/sensor" in topic: process_sensor_data(topic, payload) def init_mqtt(): """Khởi tạo kết nối MQTT""" global mqtt_client mqtt_client = mqtt.Client() mqtt_client.username_pw_set(MQTT_USER, MQTT_PASS) mqtt_client.on_connect = on_connect mqtt_client.on_message = on_message # Kết nối trong thread riêng (không block server) mqtt_client.connect(MQTT_BROKER, MQTT_PORT, 60) mqtt_client.loop_start() def send_command(api_key, device_code, value): """Gửi lệnh điều khiển đến ESP32""" topic = f"{api_key}/{device_code}" mqtt_client.publish(topic, value) print(f"📤 Sent: {topic} → {value}")

4.5. Code ESP32 (Arduino)

ESP32 subscribe topic và điều khiển relay:

// File: ESP32_SmartHome.ino #include <WiFi.h> #include <PubSubClient.h> // Cấu hình WiFi const char* ssid = "Your_WiFi"; const char* password = "password"; // Cấu hình MQTT const char* mqtt_server = "103.252.136.205"; const char* api_key = "abc123xyz"; // Lấy từ web // Chân GPIO cho relay int relay_pins[] = {2, 4, 5, 18, 19}; // v1, v2, v3, v4, v5 WiFiClient espClient; PubSubClient client(espClient); void callback(char* topic, byte* payload, unsigned int length) { // Nhận message từ server String message = ""; for (int i = 0; i < length; i++) { message += (char)payload[i]; } // Lấy device_code từ topic (VD: "abc123xyz/v1" → "v1") String topicStr = String(topic); int lastSlash = topicStr.lastIndexOf('/'); String device_code = topicStr.substring(lastSlash + 1); // Xác định relay nào int relay_index = device_code.substring(1).toInt() - 1; // "v1" → 0 // Bật/tắt relay if (message == "on") { digitalWrite(relay_pins[relay_index], HIGH); Serial.println("✅ Relay " + device_code + " ON"); } else { digitalWrite(relay_pins[relay_index], LOW); Serial.println("❌ Relay " + device_code + " OFF"); } } void setup() { Serial.begin(115200); // Setup relay pins for (int i = 0; i < 5; i++) { pinMode(relay_pins[i], OUTPUT); digitalWrite(relay_pins[i], LOW); } // Kết nối WiFi WiFi.begin(ssid, password); while (WiFi.status() != WL_CONNECTED) { delay(500); Serial.print("."); } Serial.println("WiFi connected!"); // Kết nối MQTT client.setServer(mqtt_server, 1883); client.setCallback(callback); } void reconnect() { while (!client.connected()) { if (client.connect("ESP32Client")) { // Subscribe các topic của user String topic = String(api_key) + "/+"; // abc123xyz/+ client.subscribe(topic.c_str()); Serial.println("MQTT connected!"); } else { delay(5000); } } } void loop() { if (!client.connected()) { reconnect(); } client.loop(); }

Tích hợp Face Recognition (Nhận diện khuôn mặt)

Face Recognition là gì?

Face Recognition (nhận diện khuôn mặt) là công nghệ AI có khả năng:
  • 🔍 Phát hiện (Detection): Tìm vị trí khuôn mặt trong ảnh
  • 📐 Trích xuất (Embedding): Chuyển khuôn mặt thành vector số
  • 🎯 So khớp (Matching): So sánh 2 vector để xác định cùng 1 người

Ứng dụng: Mở cửa tự động khi nhận diện đúng người đã đăng ký!

Bước 1: Face Detection

Tìm vị trí khuôn mặt trong ảnh (bounding box)

  • Input: Ảnh 640x480 pixel
  • Output: Tọa độ (x, y, width, height)
  • Thuật toán: RetinaFace (trong InsightFace)
Bước 2: Face Embedding

Chuyển khuôn mặt thành vector 512 chiều

  • Input: Ảnh khuôn mặt đã crop 112x112
  • Output: Vector 512 số thực [-1, 1]
  • Thuật toán: ArcFace (trong InsightFace)
Embedding là gì?
Embedding biến khuôn mặt thành "mã số" duy nhất. Hai ảnh cùng 1 người sẽ có embedding gần nhau, hai người khác nhau sẽ có embedding xa nhau.

[0.12, -0.34, 0.56, ..., 0.78] ← Vector 512 số đại diện cho 1 khuôn mặt
5.2. So sánh khuôn mặt (Cosine Similarity)

Để biết 2 khuôn mặt có giống nhau không, ta tính độ tương đồng cosine:

# Công thức Cosine Similarity similarity = cos(θ) = (A · B) / (|A| × |B|) # Trong đó: # A, B: 2 vector embedding # A · B: tích vô hướng # |A|, |B|: độ dài vector # Kết quả: # similarity ≈ 1.0 → Cùng 1 người (giống nhau) # similarity ≈ 0.0 → Khác người (khác nhau) # similarity < 0.4 → Chắc chắn khác người # Ngưỡng thường dùng: 0.5 - 0.6 if similarity > 0.5: print("✅ Cùng 1 người") else: print("❌ Khác người")
5.3. Quy trình "Train" (Đăng ký khuôn mặt)
Lưu ý: Với InsightFace, ta KHÔNG train model từ đầu (rất tốn thời gian). Thay vào đó, ta chỉ lưu embedding của người mới vào database.
📁 Quy trình đăng ký khuôn mặt mới 1️⃣ Chụp ảnh └── User chụp 5-10 ảnh khuôn mặt từ nhiều góc 2️⃣ Detect & Crop └── Với mỗi ảnh: tìm mặt → crop 112x112 3️⃣ Extract Embedding └── Chạy qua model → lấy vector 512 chiều 4️⃣ Lưu vào Database └── Lưu ảnh vào thư mục media/face_dataset/ └── Lưu đường dẫn vào bảng FaceImage 5️⃣ Build Index (tùy chọn) └── Tính trung bình embedding các ảnh └── Lưu vào file .pkl để nhận diện nhanh hơn
# File: face_recognition/face_utils.py - Đăng ký khuôn mặt import insightface import cv2 import numpy as np import os # Load model InsightFace (chỉ load 1 lần) face_app = insightface.app.FaceAnalysis(name='buffalo_l') face_app.prepare(ctx_id=0) # GPU=0, CPU=-1 def register_face(user_code, name, image_paths): """ Đăng ký khuôn mặt mới - user_code: Mã định danh (VD: "NV001") - name: Tên người (VD: "Nguyễn Văn A") - image_paths: Danh sách đường dẫn ảnh """ embeddings = [] for img_path in image_paths: # Đọc ảnh img = cv2.imread(img_path) # Detect khuôn mặt faces = face_app.get(img) if len(faces) == 0: print(f"⚠️ Không tìm thấy mặt trong {img_path}") continue # Lấy embedding của mặt đầu tiên embedding = faces[0].embedding # Vector 512 số embeddings.append(embedding) # Tính trung bình embedding if embeddings: avg_embedding = np.mean(embeddings, axis=0) # Lưu vào database hoặc file save_embedding(user_code, name, avg_embedding) print(f"✅ Đã đăng ký {name} với {len(embeddings)} ảnh") else: print("❌ Không có ảnh hợp lệ")
5.4. Quy trình nhận diện
📁 Quy trình nhận diện khuôn mặt 1️⃣ Nhận ảnh từ camera └── ESP32-CAM gửi frame qua MJPEG stream 2️⃣ Detect khuôn mặt └── Tìm tất cả khuôn mặt trong frame 3️⃣ Extract Embedding └── Chuyển thành vector 512 chiều 4️⃣ So sánh với Database └── Tính cosine similarity với từng người đã đăng ký └── Tìm người có similarity cao nhất 5️⃣ Kiểm tra ngưỡng └── similarity > 0.5 → Nhận diện thành công └── similarity ≤ 0.5 → Không xác định 6️⃣ Thực hiện hành động └── Gửi MQTT mở cửa └── Lưu log vào AttendanceLog
# File: face_recognition/face_utils.py - Nhận diện def recognize_face(image): """ Nhận diện khuôn mặt từ ảnh Returns: (user_code, name, confidence) hoặc None """ # Detect khuôn mặt faces = face_app.get(image) if len(faces) == 0: return None # Lấy embedding của mặt phát hiện được query_embedding = faces[0].embedding # Load tất cả embedding đã đăng ký từ database registered_faces = load_all_embeddings() best_match = None best_score = 0 for user_code, name, db_embedding in registered_faces: # Tính cosine similarity score = cosine_similarity(query_embedding, db_embedding) if score > best_score: best_score = score best_match = (user_code, name) # Kiểm tra ngưỡng THRESHOLD = 0.5 if best_score > THRESHOLD: user_code, name = best_match print(f"✅ Nhận diện: {name} (confidence: {best_score:.2f})") # Mở cửa qua MQTT send_door_open_command() return (user_code, name, best_score) print(f"❌ Không nhận diện được (best: {best_score:.2f})") return None def cosine_similarity(a, b): """Tính độ tương đồng cosine giữa 2 vector""" return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
